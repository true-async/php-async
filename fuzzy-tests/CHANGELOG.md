# Fuzzy-tests changelog

Test-coverage and harness changes for `ext/async/fuzzy-tests/`. Kept
separate from the main `CHANGELOG.md` so user-facing extension features
are not mixed with test-suite history.

## Unreleased

### Added
- **#138 Layer 1 IO chaos — concurrent file writes** — `io/file_concurrent_writes.feature` (4 scenarios → 6 .phpt). Regression backstop for `php_stdiop_write` (#129/#133): every coroutine writing the SAME file descriptor parks on its single shared async-IO event; a write completing for any coroutine wakes them all, and a spuriously-woken coroutine must re-suspend until its own request completed — before the fix it disposed the in-flight libuv request and writes were silently lost on macOS/Windows. `tests/io/083` only pins a deterministic 4-worker scenario; this feature crosses worker count, chunk size, iteration count, and interleaved sleeps under random ChaosNet scheduling. New `Context::$files` registry + steps `Given a shared file "F"`, `coroutine "X" writes N chunks of M bytes to shared file "F"`, `coroutine "X" closes shared file "F"`, and variadic invariant `shared file "F" byte size equals counter "A" plus counter "B" ...` — the kernel-observed file size must equal the sum of bytes every coroutine claims fwrite() accepted, exactly the assertion the spurious-wakeup bug broke. Tempfile lifecycle: handle flush+close in run() teardown so Then assertions can filesize() it, path unlink deferred to `Context::__destruct()`. 6 generated .phpt × 10 seeds green.
- **#138 Layer 1 IO chaos — IPv6 connect-watcher** — `io/connect_v6.feature` (3 scenarios → 3 .phpt). Same race surface as the v4 features but on an AF_INET6 socket; uses `[::ffff:192.0.2.1]:81` (v4-mapped TEST-NET-1) so the v6 socket-creation branch is exercised while the SYN still goes out to a routable address and the connect actually parks. Includes a v4+v6 mixed scenario to catch family-specific bugs in the shared `network_async_await_stream_socket()` release path. New step `coroutine "X" connects to IPv6 blackhole "ADDR" with timeout N ms` and SKIPIF tag `tcp-blackhole-v6` (skips hosts without v6 egress).
- **#138 Layer 1 IO chaos — connect-watcher** — `io/cancel_during_connect.feature` (3 scenarios → 8 .phpt) and `io/connect_with_timeout.feature` (3 scenarios → 5 .phpt). Exercises `network_async_await_stream_socket()` (the reactor's connect-watcher) via `stream_socket_client()` against an RFC 5737 TEST-NET-1 blackhole (`192.0.2.1:81`): SYN goes out, no response, kernel returns EINPROGRESS, the connect parks in poll. Killer cancel mid-wait and explicit short-timeout fire — both must release the watcher without UAF or libuv handle leak. New step `coroutine "X" connects to TCP blackhole "ADDR" with timeout N ms` with the `io_connect_{attempts,ok,timeout,cancelled,failed}` outcome family, plus new generic step `counter "X" is at least N` for dominant-bucket assertions. New SKIPIF probe `tcp-blackhole`: sanity-checks that the chosen blackhole IP actually suspends past 50 ms on this host — Alpine/CI hosts without a default route would otherwise fail synchronously with ENETUNREACH and never engage the connect-watcher, silently lying about coverage. Connect cancel timeouts capped at 5 s (was 30 s) so a regression in the cancel path fails fast instead of hanging. Every cancel/timeout scenario now asserts `io_connect_{cancelled,timeout}` ≥ 1 — turns the sum-only invariant into a real check that the targeted code path actually fired. 13 generated .phpt × 8 seeds green (WSL2 dt≈50 ms for a 50 ms timeout confirms the watcher is engaged).
- **#138 Layer 1 IO chaos — close & concurrent-readers** — `io/stream_close_during_read.feature` and `io/concurrent_readers.feature` (5 scenarios incl. 4-reader cancel-half + delay-varied outline → 20 .phpt). Cross-coroutine `fclose()` of the write end races a parked `fread()`; N coroutines park on one shared `stream_socket_pair`; cancellation of half the readers must leave the survivors unaffected. New `Context::pipes` registry + steps: `Given a shared pipe "P"`, `coroutine "X" reads from shared pipe "P"`, `coroutine "X" writes "<msg>" to shared pipe "P"`, `coroutine "X" closes the write end of shared pipe "P"`. **Found a real bug** — cancelling one of N readers on a shared poll proxy double-stopped the proxy and parked the survivors forever; fixed in php-src `Zend/zend_async_API.{c,h}` and `ext/async/scheduler.c` (ABI bumped to v0.18.0). See `FINDINGS.md`.

- **#136 SQLite pool-focused coverage** — `db/sqlite_chaos.feature` (6 scenarios). Local file, no Toxiproxy, pdo_sqlite operations do not yield — exercises only the PDO connection pool itself (per-coroutine `sqlite3*` slots over one shared file). New steps `a [pooled] SQLite database "DB"`. Schema seeded per-scenario into a per-PID file under `sys_get_temp_dir()`, removed in tearDown. Only `pdo_sqlite` required by SKIPIF.
- **#136 Concurrent transactions and heterogeneous workloads** — `db/concurrent.feature` (9 scenarios → 22 .phpt) targets many coroutines doing different things at once on one pooled `$pdo`: concurrent transactions, writer racing reader (reader's `WHERE id<=5` snapshot is interleaving-independent of writer's `id>5` insert), mixed workloads, a transaction cancelled while a sibling reader keeps working, a storm on an undersized pool. `StandardSteps::dbTransaction` runs a multi-statement transaction now (BEGIN → INSERT → read-back SELECT → COMMIT) so each yields several times. **Found a UAF in the PDO connection pool** under coroutine cancellation mid-transaction — fixed in php-src `ext/pdo/pdo_pool.c`.
- **#136 PgSQL coverage** — `db/pgsql_chaos.feature` (12 scenarios → 30 .phpt) mirrors `mysql_chaos.feature` against a Toxiproxy-fronted PostgreSQL server. `ChaosNet::openDbConnection()` and `dbUpstream()` driver-aware (mysql / pgsql); slow-query step picks `SELECT pg_sleep(2)` vs `SELECT SLEEP(2)` at action-run time. New steps `a [pooled] PgSQL database "DB"`. Toxiproxy and PDO client steps relaxed to driver-agnostic `requires`; the database declaration carries the driver requirement. `generate.php` gains `pdo_pgsql` and `pgsql-server` SKIPIF probes.
- **#136 mysqli coverage** — `db/mysqli_chaos.feature` (10 scenarios) — same Toxiproxy-fronted MySQL through the mysqli extension. mysqli has no pool, every chaos query opens its own connection. New steps `a MySQLi database "DB"` and `coroutine "C" queries|runs a slow query via|runs a transaction via mysqli "DB"`. `ChaosNet::openMysqliConnection()` with `MYSQLI_REPORT_STRICT`. Toxiproxy toxic steps driver-agnostic now.
- **#136 PDO MySQL coverage** — `db/mysql_chaos.feature` (12 scenarios → 30 .phpt). Async PDO MySQL through Toxiproxy: latency, bandwidth, TCP slicer, `reset_peer` mid-query / mid-transaction. Pooled and non-pooled. `ChaosNet` is the new network-fixture layer extracted from `Context`; new steps `a [pooled] MySQL database "DB"` / `Toxiproxy adds latency|throttles|slices|resets database "DB"` / `coroutine "C" queries|runs a slow query on|runs a transaction on database "DB"`. Env-driven connection (`CHAOS_MYSQL`, `CHAOS_MYSQL_USER/PASS/DB`). Opt-in via SKIPIF (ext/pdo_mysql + reachable MySQL).
- **#136 HTTP / CURL coverage** — `EvilPeer::serveHttp` (HTTP/1.1 evil server mode) + `curl/http_chaos.feature` (12 scenarios → 35 .phpt). Reactor-driven `ext/curl` client under every body toxic plus HTTP-specific ones — chunked TE, mendacious Content-Length, dribbled headers, arbitrary status. Killer-cancel mid-transfer covered. New step `coroutine "C" fetches peer "EP" over HTTP`. **Found a real bug** — async curl dropped all but the first chunk of a chunked response body (`CURLE_WRITE_ERROR`); fixed in php-src `ext/curl/curl_async.c`. See `FINDINGS.md`.
